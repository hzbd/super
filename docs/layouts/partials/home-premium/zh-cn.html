订阅插件 💎 授权版

所有人使用同一个 superd 二进制。可选能力以签名插件形式交付 — 将供应商提供的 [license].key 写入 conf/super.toml，并把授权库文件放入 $SUPER_ROOT/plugins/。运行时仅加载密钥中列出的模块。

预发布状态：订阅交付仍在开发中，暂不对客户开放。下文描述目标模型。
1
配置 License
将供应商提供的密钥写入 conf/super.toml 的 [license] 段 — 决定启用哪些模块及到期时间
2
部署插件库
将订阅交付包中的文件复制到 $SUPER_ROOT/plugins/
3
重启 superd
服务启动时自动校验密钥并启用对应功能 — 无需更换核心程序

安全与治理

plugins/security.so
  • API 路由 Bearer 令牌认证
  • RBAC 角色 — admin、operator、viewer
  • 不可篡改的操作审计日志
认证 访问控制 审计

事件通知

plugins/notify.so
  • Webhook 通道 — Slack、钉钉、飞书、自定义 URL
  • process_fatal 等系统事件推送
  • 通过 conf/notify.toml 热更新
事件通知

资源隔离

plugins/isolation.so
  • cgroups v2 按程序限制 CPU / 内存
  • 热更新限额，无需重启进程
  • CLI --cpu / --memory 强制生效

🐧 仅适用于 Linux — cgroup 隔离不支持 macOS、Windows 或 FreeBSD。
