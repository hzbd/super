Official Plugins 💎 Commercial

One superd binary for everyone. Commercial capabilities ship as signed plugins — drop authorized plugins/*.so under $SUPER_ROOT and add a valid [license].key to conf/super.toml. Only plugins listed in the license are loaded at runtime.

Pre-release: The plugin subscription model is under active development and not offered for customer delivery yet. Documentation describes the target architecture.
1
Configure license
Paste the vendor-supplied key into [license] in conf/super.toml — defines which plugins are active and when they expire
2
Deploy plugin files
Copy delivered plugin files into $SUPER_ROOT/plugins/ (e.g. security.so, ui.so)
3
Restart superd
On startup the service validates the license and enables matching features — no need to replace the core binary

Resource Isolation

  • cgroups v2 CPU and memory limits per program
  • Hot-update limits without restarting the process
  • CLI --cpu / --memory enforcement

🐧 Linux only — cgroup isolation is not available on macOS, Windows, or FreeBSD. There is no isolation.dylib.
